# 功能开发清单 (Feature List)

> 文档说明：本文档为"浙大晨兴文化中国"项目的核心功能清单。
>
> 核心策略：采用"混合内容策略 (Hybrid Content Strategy)"。核心静态内容本地化，时效性内容（新闻、活动、随笔）通过"外部链接引用"模式指向微信公众号。
>
> **多语言支持**：项目将支持中英双语版本。当前阶段优先完成中文版本的开发，英文版本将在中文版本稳定后实施。中文版本为默认版本，无需在 URL 中添加语言前缀。

## 2. 功能模块清单 (Feature Modules)

### 2.0 首页 (Homepage)

> **定位** : 品牌形象的集中展示窗口，结构上采用"首屏视觉+三大核心板块"的布局。

* **首屏：视觉滚动横幅 (Hero Carousel)**
  * **内容** : 循环展示"近期活动剪影"的高清图片（如年度论坛现场、访学合影）。
  * **视觉** : 全屏或宽幅布局，叠加半透明蒙层。
  * **元素** : 核心 Slogan（如"视域·情感·观点"）
* **板块一：文化中国介绍 (Project Overview)**
  * **定位** : "我是谁"。快速传递项目价值。
  * **内容** : 提炼后的使命背景与培养宗旨摘要。
  * **交互** : 配以极简的排版和"了解更多"按钮，引导至【计划介绍】详情页。
* **板块二：近期活动动态 (Latest Updates)**
  * **定位** : "正在发生什么"。展示项目的活跃度。
  * **内容** : 聚合【课程教学-新闻场记】与【特色活动】中的最新内容。
  * **形式** : 卡片流或时间轴布局。
  * **交互** : 点击卡片直接跳转至对应的微信公众号推文。
* **板块三：校友与师资 (Alumni & Community)**
  * **定位** : "我们的社群"。展示人才密度与底蕴。
  * **内容** :
  * **师资墙** : 导师头像的横向滚动展示。
  * **校友高光** : 展示优秀校友代表或学员数据（如：已培养 XX 期，覆盖 XX 行业）。
  * **交互** : 提供入口跳转至【学员风采】或【计划介绍-师资嘉宾】。
* **全局导航与页脚**
  * 磨砂质感悬浮导航栏。
  * 包含公众号二维码、联系方式的功能性页脚。

### 2.1 模块一：计划介绍 (Program Introduction)

> **定位** : 静态展示，强调极致的图文排版。

* **使命背景** : 项目历史与愿景介绍。
* **培养宗旨** : 核心教育理念阐述。
* **师资嘉宾** : 导师名录（卡片墙），支持点击查看详情。
* **儒商中心** : 静态介绍页。
* **周老师专栏** : 专栏文章列表（本地存储或精选链接）。

### 2.2 模块二：学员风采 (Alumni Showcase)

> **定位** : 混合模式，建立校友归属感。

* **各期学员** :
* 按届次（如：第一期、第二期）筛选的学员名录。
* **校友故事/随笔** :
* 一部分为微信公众号链接跳转，一部分为固定blog

### 2.3 模块三：课程教学 (Curriculum)

> **定位** : 历史资料归档，轻量化展示。

* **新闻场记** : 课程活动的时间轴或列表展示（跳转公众号）。
* **课程介绍** : 按年份归档的历史课程大纲（跳转公众号）。

### 2.4 模块四：特色活动 (Featured Activities)

> **定位** : 展现项目活力与对外交流成果。

* **年度论坛** : 重点视觉展示，论坛集锦回顾（跳转公众号）。
* **访学交流** : 精彩瞬间图库（Gallery 模式，跳转公众号）。
* **其他活动** : 标准列表展示。

### 2.5 模块五：招生信息 (Admissions)

> **定位** : 极简信息引导，转化流量至私域。

* **基本介绍** : 招生要求、流程与时间表 (Timeline)。
* **关注引导** : 醒目的公众号二维码与咨询引导区域。

### 2.6 模块六：运维后台 (Operations Admin)

> **定位** : 后台内容管理中心。

* **资源管理 (Resource CMS)**
  * 对全站"外部链接资源"（新闻、故事、活动）的统一增删改查。
  * 置顶/推荐管理：控制首页显示哪些内容。
* **学员/师资库管理**
  * 结构化数据的批量录入与编辑。
* **外部资源录入 (Manual Resource Entry)**
  * **功能** : 提供基础表单，供管理员手动输入公众号文章 URL、标题及简介。

### 2.7 模块七：多语言支持 (Internationalization / i18n)

> **定位** : 提供中英双语版本，面向国际化受众。
> **开发策略** : 分阶段实施，优先完成中文版本，英文版本后续开发。

* **语言版本规划**
  * **中文版本（默认）** : 当前开发重点，所有功能优先在中文版本中实现。
  * **英文版本** : 待中文版本稳定后，基于 Next.js i18n 路由（如 `/en/*`）实现。
* **内容翻译策略**
  * **静态内容** : 核心页面（使命背景、培养宗旨等）需要提供中英双语内容。
  * **动态内容** : 数据库中的内容字段支持中英双语存储（如 `title_zh`, `title_en`）。
  * **外部链接** : 微信公众号文章以中文为主，英文版本可提供摘要翻译或保持原链接。
* **技术实现**（待 Phase 5 实施）
  * 使用 Next.js App Router 的国际化路由（`[locale]` 动态段）。
  * 或使用 `next-intl` 等 i18n 库进行内容管理。
  * 语言切换器：在导航栏提供中英文切换按钮。
